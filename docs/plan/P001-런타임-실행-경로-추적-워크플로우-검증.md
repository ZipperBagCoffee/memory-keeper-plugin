# P001 - 런타임 실행 검증을 워크플로우 검증에 추가

## Intent (의도)
검증의 정의를 재정의한다. 현재 검증은 비평 + 의도부합 + 코드존재확인만 하고 "런타임에서 실제로 돌아가는가"를 검증하지 않는다. 이 4번째 요소를 Work Agent, Review Agent, Orchestrator 모든 레이어에 의무화한다. D001 결론 기반.

## Scope (범위)
포함:
- skills/workflow/SKILL.md — Phase 8, 9, 10에 런타임 실행 검증 절차 추가
- skills/workflow/SKILL.md — "What Each Role Needs" 테이블 업데이트
- skills/workflow/SKILL.md — Anti-pattern 추가
- scripts/inject-rules.js — RULES의 "Verification standard" 규칙 강화

제외:
- 새 파일 생성 없음
- 다른 스킬 파일 변경 없음
- 버전 범프 (별도 작업)

## Plan (계획)

- [ ] 단계 1: SKILL.md Phase 8 (Implement) — Work Agent가 구현 후 자기 코드의 런타임 실행 경로를 추적하는 절차 추가
- [ ] 단계 2: SKILL.md Phase 9 (Verify) — 기존 "Trace execution path" 한 줄을 구체적 절차로 확장. 런타임 실행 검증 포맷 정의
- [ ] 단계 3: SKILL.md Phase 10 (Meta-Verify) — Orchestrator가 런타임 실행 검증을 최종 검증하는 절차 추가. Self-enforcement Checklist에 항목 추가
- [ ] 단계 4: SKILL.md "What Each Role Needs" 테이블 — 3개 역할 모두에 런타임 실행 검증 책임 추가

- [ ] 단계 6: inject-rules.js — "Verification standard" 규칙에 런타임 실행 검증 의무화 추가

## Verification Criteria (검증 기준)
1. Phase 8에 Work Agent 런타임 실행 검증 절차가 존재하고, "구현 후 실행 경로를 추적하라"는 의무가 명시되어 있다
2. Phase 9에 런타임 실행 검증 포맷이 정의되어 있고, Review Agent가 이를 검증하는 절차가 있다
3. Phase 10에 Orchestrator가 런타임 실행 검증 결과를 확인하는 항목이 Self-enforcement Checklist에 있다
4. "What Each Role Needs" 테이블의 3개 역할 모두에 런타임 실행 검증 관련 내용이 포함되어 있다
5. Anti-pattern 목록에 런타임 실행 검증 관련 항목이 있다
6. inject-rules.js RULES에 런타임 실행 검증이 verification standard의 일부로 명시되어 있다
7. **런타임 검증:** 이 SKILL.md를 사용하는 워크플로우를 mental execution으로 돌려봤을 때, Phase 8→9→10 흐름에서 런타임 실행 검증을 빠뜨릴 수 없는 구조인가
